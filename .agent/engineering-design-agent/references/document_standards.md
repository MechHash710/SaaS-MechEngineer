# Document Standards - Engineering (Brazil)

## Estrutura de Documentos Técnicos
Todos os documentos gerados pela plataforma devem seguir a hierarquia padrão:

1. **Cabeçalho Técnico:**
   - Logotipo da Empresa (alinhado à esquerda)
   - Título do Documento (Centralizado, Bold)
   - Identificador (Número/ID) + Data de Emissão

2. **Identificação do Projeto:**
   - Cliente: Nome / Endereço / CNPJ
   - Projeto: Descrição Curta
   - Responsável Técnico: Nome + Título (Engenheiro) + CREA/CFT

3. **Cálculos e Premissas:**
   - Devem usar a fonte mono-espaçada para tabelas de steps.
   - Cada step deve ter a norma de referência ao lado.

4. **Rodapé Normativo:**
   - Lista resumida de normas aplicadas (NBR, ASHRAE, etc.)
   - Paginação: "Página X de Y"

## Regras para Nota Fiscal de Serviço (NFS-e)
- **Prestador:** Dados completos da empresa de engenharia.
- **Tomador:** Dados do cliente.
- **Serviço:** Descrição clara (ex: "Consultoria em Eficiência Energética e Climatização").
- **Tributação:** ISS (padrão 5%), PIS, COFINS destacados se aplicável.
